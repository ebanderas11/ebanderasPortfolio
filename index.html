Reflections

User Experience

One of the most important topics we covered in our Front End Web Dev class is User Experience. The way users interact with the website and how easy it is to use overall will ultimately determine if you website will be succesful in retaining users or drive them away.

It is important to consider all users when creating your website, but I believe it is more important to find the audience you are catering to and design a website based off of that. Of course I want my site to be accessable by all, but at the end of the day it is my target audience that will decide if they will use the site or not.

It is important to look up your user demographic and understand user habits. This way you can plan ahead and build accordingly. It is easy to do the research in the beginning as opposed to after your site is published.

Responsive Design

Today most of use our phones for daily tasks that we would normally do on a computer. Most website users today access websites from their phones. That's why it is especially important to make sure your website is mobile responsive.

Personally, I like to build my websites with a mobile first approach. I build everything out starting off with how it will look on a mobile device. Once the site looks good on mobile, I start to scale the site to bigger screen sizes such as tablets or computer monitors.

A mobile responsive site can be achieved using tools such as flex-box and media queries. I like to combine the two to create clean and responsive sites.

Technical Documentation Page for React

Using repl.it we modified JSON and used AJAX to create a single page application that basically updates on the same page. Whenever you click for more info to come up, the info is added to the current page instead of having a new page open for the specific info. It also allows for the page to be more compressed and have less clutter.

Overall this was a good introduction to JSON/AJAX. It helped understand why using JS and JSON together is so useful.
